# Analysis of Flows and PCAPs: Behavioral Signatures in NetFlow and DNS

This directory contains scripts used in the thesis to analyze NetFlow-like flow data and PCAP traces, focusing on behavioral signatures in DNS and TLS traffic.

All scripts:

- Use dependencies pinned in `uv.lock`
- Write their results to the `output/` directory 
- Are intended to be run via [`uv`](https://github.com/astral-sh/uv), e.g. `uv run script.py ...`


## Datasets
See the dataset description and acquisition instructions in the [main README](../README.md).

## Data sources

Datasources used in these analysis:
- `all/filtered_flows.txt` -> A tranalyzer2 flows file, generated from all of the PCAP's from the interceptor
- `dns/merged_output.pcap` -> A merged pcap file from all of the PCAP's generated by the DNS authoritative server
- `receiver/merged_output.pcap` -> A merged pcap file from all of the PCAP's generated by the webserver serving our experiments

## Script overview
| Script                 | Label             | Primary purpose                                                |
| ---------------------- | ----------------- | -------------------------------------------------------------- |
| `dns_authoritative_asn_summary.py`          | Authoritative DNS | Analyze DNS server PCAP and aggregate queries by source ASN    |
| `dns_top_queried_domains.py`           | Interceptor       | Find the most frequently queried domains in flow data          |
| `sni_domains_without_dns.py`          | Interceptor       | Compare DNS and SNI domains; list domains only seen in SNI     |
| `receiver_ttl_distribution.py`      | Webserver         | Inspect TTL distribution of received packets in PCAP           |
| `hourly_traffic_by_experiment.py`         | Interceptor       | Visualize traffic volume per hour of day across experiments    |
| `interceptor_full_flow_analysis.py`  | Interceptor       | Main analysis script: DNS/TLS volume, ratios, TLDs, top talkers|

Below you will find details, parameters, and example usage for each script, as well as how they were used in the thesis.

## `dns_authoritative_asn_summary.py` - Authoritative DNS

**Label:** Authoritative DNS  
**Role in thesis:** Used to determine where DNS queries to the authoritative server originate from (by ASN), to provide context on who is querying the domain(s).

### Inputs & parameters

```text
dns_authoritative_asn_summary.py <pcap_path> <ip_to_asn_csv>
```

- `pcap_path`: Path to the DNS server PCAP (e.g., `dns/merged_output.pcap`).
- `ip_to_asn_csv`: Path to the IP→ASN mapping CSV (e.g., `dbip-asn-lite-2026-04.csv`).

### Output
- A table listing, for each ASN (and/or IP range), how many DNS queries were observed.  
- Useful to identify which networks generate most DNS traffic to the authoritative server.

### Example

```sh
uv run dns_authoritative_asn_summary.py dns/merged_output.pcap /home/thies/project/resip/data/dbip-asn-lite-2026-04.csv
```

## `dns_top_queried_domains.py` - Most queried domains

**Label:** Interceptor  
**Role in thesis:** Used to quickly inspect which domains are most frequently queried in the intercepted DNS traffic (e.g. to show the prominence of `api.pawns.app`).

### Inputs & parameters

```text
dns_top_queried_domains.py <flow_file>
```

- `flow_file`: Path to a flow file with DNS information (e.g., `all/filtered_flows.txt`).

### Output

- Plain-text list of domains with corresponding query counts, printed to stdout.
- Example output line:

  ```text
  17919  api.pawns.app
  ```

- Used mainly for quick, contextual inspection (not for generating figures).

### Example

```sh
uv run dns_top_queried_domains.py all/filtered_flows.txt
```


## `sni_domains_without_dns.py` - Domains only seen in SNI

**Label:** Interceptor  
**Role in thesis:** Used to identify domains that appear only in TLS SNI and not in DNS queries, showing traffic that bypasses DNS visibility.

### Inputs & parameters

```text
sni_domains_without_dns.py <flow_file>
```

- `flow_file`: Path to the flow file that includes both DNS and TLS/SNI information (e.g., `all/filtered_flows.txt`).

### Output

- Plain-text listing (by day) of domains that are **not** observed in DNS but **are** observed in SNI.
- Printed to stdout, used for contextual analysis in the thesis.

### Example

```sh
uv run sni_domains_without_dns.py all/filtered_flows.txt
```

## `receiver_ttl_distribution.py` - TTL analysis of received packets

**Label:** Webserver  
**Role in thesis:** Used to inspect the Time-To-Live (TTL) distribution of packets arriving at the receiver during experiment periods.

### Inputs & parameters

```text
receiver_ttl_distribution.py <pcap_path>
```

- `pcap_path`: Path to the receiver PCAP (e.g., `receiver/merged_output.pcap`).

### Output

- Plain-text summary printed to stdout with:
  - TTL values
  - Number of packets per TTL
  - Percentage of total packets
  - Top source IPs per TTL

Example output:

```text
TTL 247:    3 packets    (  0.04 % of all)
  Top 3 IPs for TTL 247:
    1. 1.2.3.4    1 packets    ( 33.33 % of TTL 247)
    2. 1.2.3.5    1 packets    ( 33.33 % of TTL 247)
    3. 1.2.3.6    1 packets    ( 33.33 % of TTL 247)
```

### Example

```sh
uv run receiver_ttl_distribution.py receiver/merged_output.pcap
```

## `hourly_traffic_by_experiment.py` - Traffic per hour of day

**Label:** Interceptor  
**Role in thesis:** Used to visualize how traffic volume varies over the day for both experiment phases (e.g., EarnApp vs Pawns), and to quantify total data usage per phase.


### Inputs & parameters

```text
hourly_traffic_by_experiment.py <flow_file>
```

- `flow_file`: Path to the flow file with experiment phase labeling (e.g., `all/filtered_flows.txt`).

### Output

Written to `output/`:

- `traffic_per_hour_earnapp.png` — Traffic per hour for the EarnApp phase
- `traffic_per_hour_pawns.png` — Traffic per hour for the Pawns phase
- `data_usage_summary.txt` — Text summary of data usage per experiment phase

### Example

```sh
uv run hourly_traffic_by_experiment.py all/filtered_flows.txt
```

## `interceptor_full_flow_analysis.py` - Main interceptor analysis

**Label:** Interceptor  
**Role in thesis:** Main script for analyzing intercepted flows. Used to generate most of the figures and tables about DNS/TLS behavior and traffic characteristics.

### Inputs & parameters

```text
interceptor_full_flow_analysis.py <flow_file>
```

- `flow_file`: Path to the main flow file with DNS, TLS, and experiment phase information (e.g., `all/filtered_flows.txt`).

### Output

All outputs are written into `output/` and include:

- `dns_activity_over_time.png`  
  DNS query counts/volume over time, split by experiment phases.
- `tls_flows_over_time.png`  
  TLS flows or bytes over time.
- `traffic_and_ratio_over_time.png`  
  Aggregate traffic and DNS/TLS ratio over time.
- `protocol_distribution_pie.png`  
  Pie chart of protocol distribution.
- `dns_tld_distribution_before_after.png`  
  Comparison of DNS TLD distribution before vs after the experiment change.
- `sni_tld_distribution_before_after.png`  
  Comparison of SNI TLD distribution before vs after the experiment change.
- `top_talkers_tables.tex`  
  LaTeX tables listing the largest application-layer “talkers” (e.g., top domains or endpoints by volume).

These outputs correspond to the main figures and LaTeX tables used in the analysis chapters.

### Example

```sh
uv run interceptor_full_flow_analysis.py all/filtered_flows.txt
```
